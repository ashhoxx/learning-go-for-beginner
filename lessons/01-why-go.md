# Урок 1: Почему Go — не «ещё один язык»

## Первый запуск

Создай файл `main.go` и вставь этот код:

```go
package main

import "fmt"

func main() {
    fmt.Println("Привет, мир!")
}
```

Запусти в терминале:
```bash
go run main.go
```

Если видишь `Привет, мир!` — ты запустил программу на Go.

---

## Что делает Go особенным?

### 1. Компилируется в один бинарник
- Не нужен интерпретатор или виртуальная машина.
- Программа — это один исполняемый файл. Можно скопировать на сервер и запустить.

### 2. Используется в реальных системах
- Docker, Kubernetes, Terraform — написаны на Go.
- Go выбирают, когда нужна простота, скорость и надёжность.

### 3. Намеренные ограничения
- Нет наследования → меньше сложных иерархий.
- Нет исключений → ошибки обрабатываются явно.
- Нет неявного приведения типов → меньше скрытых багов.

Это не «бедность языка» — это **защита от ошибок**.
